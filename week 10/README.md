# MODEL VIEW PATTERNS

---

# WHAT I UNDERSTAND

Model View patterns means we break application into parts:

- Model → data (like user data, tasks)  
- View → what user see on screen  
- Logic part → connect data and screen  

This help to keep code clean and easy.

---

# MODEL VIEW CONTROLLER 

In this:

- One part store data  
- One part show screen  
- One part control everything  

Example:  
User click button → control part work → data change → screen update  

My thought:  
This is easy and simple.

---

# MODEL VIEW VIEWMODEL 

In this:

- There is one middle part  
- It connect data and screen  
- It update screen automatically  

So we don’t need to update screen again and again.

My thought:  
Little confusing for me at start.

---

# MODEL VIEW PRESENTER 

In this:

- One part handle all logic  
- Screen is simple  
- Data is separate  

That middle part control everything.

My thought:  
More work and more code.
